## v1.8.0 - 2026-07-22

### Windows 缩放稳定性

- Windows 分层窗口改为通过一次 `UpdateLayeredWindow` 原子提交新位图、位置和尺寸，不再先把旧位图拉伸到新窗口，修复滚轮缩放时的抖动和闪烁。
- 滚轮缩放不再反复创建会改变窗口宽度的提示气泡；右键菜单调整大小仍保留一次确认提示。
- 新增 `tools/test_windows_layered_resize.ps1`，防止提前 `SetBounds` 的旧问题回归。

### 不可打断的互动队列

- Windows、macOS 和 Web 的当前动作都必须完整播放，快速点击最多预排一个动作，不会直接切掉当前动画。
- 动作结束后先显示约 100ms 的自然站立交接帧，再开始队列中的下一动作。
- 首次排队和队列已满都有中英文对白提示；英文模式提示保持纯英文。

### Coco Web Pet

- 新增独立 `web/` 静态 PWA，复用 32×8 动作帧和 5×7 待机换装帧，可单独部署到 GitHub Pages。
- 增加动作点播、七区域点击、拖动、滚轮缩放、随机/自动表演、待机和对白开关、换装、四种背景、暂停、全屏与键盘快捷键。
- 设置保存在浏览器本地；Service Worker 按使用情况缓存动画帧，避免首次打开就下载所有资源。
- 新增 Pages 部署工作流、本地预览脚本和 Web 资源/逻辑校验。

## v1.7.0 - 2026-07-22

### 自主表演与安静站立

- 待机不再以 115ms 的速度无限循环整套帧；默认保持自然站立，每隔约 3–8 秒才播放一次完整的待机小动作。
- 无操作约 18–36 秒后，Coco 会从一组自然动作中随机表演一次，并有 50% 概率显示对白。
- 鼠标操作会延后自动表演，拖动过程中不会突然触发动作。
- Windows 与 macOS 使用一致的时间线，并新增自动待机行为测试。

## v1.6.0 - 2026-07-22

- Windows 与 macOS 默认使用英文；仅当系统 UI 语言为中文（包括简体、繁体及地区变体）时默认启用中文。
- 非中文系统固定使用纯英文，右键菜单不显示语言切换，窗口标题、菜单、提示、错误和对白均无可见中文。

- Windows 单文件产物改为 `CocoDesktopPet.exe`；macOS 默认包名改为 `Coco Desktop Pet.app`，中文显示名称由系统本地化资源提供。
- 新增 Windows 双 Culture 行为测试，并扩展 macOS 静态本地化检查。

## v1.5.1 - 2026-07-22

### 网格渗漏清理，修复头顶出现其他动作的脚

- 修复根因：创作阶段的 3×2 生成表中，部分角色画得比 512 像素格子略高，脚或头顶越过网格线渗入相邻格子。固定网格切割后，这些碎块悬在相邻帧的顶部或底部——最明显的是左探头（14）、点头（05）等动作头顶出现上排角色的两只脚；摇摆（06）、旋转（07）、困倦（32）、八字移动（16）也有较小的同类渗漏，压扁（02）、缩小（13）、跳舞（22）、眩晕（25）则有反方向的底边碎块。
- `tools/prepare_frame_animation_v2.py` 切帧时新增 `remove_grid_bleed`：从格子上下边缘做泛洪填充，清除「面积 ≤4000 像素且距边缘深度 ≤48 像素、与角色主体不连通」的碎块。真实的高大姿态是 8~10 万像素的单一连通块，不会被误删；清理后全量扫描确认零残留。

### 帧位置对齐，修复持续晃动

- 修复根因：创作阶段的 3×2 生成表中，角色在每个 512×512 格子里的位置系统性漂移（待机呼吸帧整体横移 25~34 像素，点头等动作的水平中心乱跳可达 ±110 像素）。运行时每帧固定绘制在同一矩形，帧内位置漂移直接表现为角色在屏幕上晃动。
- `tools/prepare_frame_animation_v2.py` 切帧后新增脚部锚点对齐：以 `neutral_512.png` alpha 底部条带的加权质心为基准，水平平移每个中间帧使脚部对齐；使用脚部而非全身质心，避免伸手、倾斜等姿态拉偏对齐结果。
- 待机帧额外将底边钉到 neutral 底边（呼吸时脚不离地）；动作帧垂直方向保留原始数据，跳跃、空翻的离地弧线不受影响。
- 实测效果：待机水平漂移由 33 像素收敛至 ±1 像素，点头由 ±110 像素收敛至 ±2 像素，跳跃垂直弧线（509→421→509）完整保留。
- 左右跳、八字移动、太空步、潜行和冲锋使用独立的平滑整体轨迹；轨迹首尾归零，所以既保留脚锚去抖，也不会把有方向的动作变成原地摆姿势。
- 全部 292 个逻辑帧重新生成，Windows/macOS 共用同一套对齐后资源；帧资源、连续性、点击区域、无头冒烟与 macOS 静态检查全部通过。

### 无损减小安装包

- 32 个动作共用同一张首尾站立帧，5 套待机循环复用自己的首帧；运行时归档由 292 张 PNG 减为 222 张互不重复的 PNG，启动时以共享引用还原完整时间线。
- 不降低 512×512 分辨率、不量化颜色、不改变透明度或麻布纹理；Windows 与 macOS 都采用同一套无损去重规则。
- 修复 macOS `CFBundleDisplayName` 与 `CFBundleName` 的错误编码，恢复为 `Coco桌宠`。

## v1.5.0 - 2026-07-21

### 整帧动画重制

- 废弃生产环境中的头部、手臂、腿和服装运行时拼接，Windows 与 macOS 每次只绘制一张完整 Coco 帧。
- 新增自然站立待机：双臂放下、双脚着地，并持续呼吸、眨眼和轻微手脚活动，不再把原始招手参考图当作待机。
- 为 32 个互动分别编写分镜并分别生成 8 帧动画；32 个中间关键姿势互不重复。
- 所有互动第 1/8 帧与默认待机首帧逐像素一致，动作结束后直接回到待机，无交叉淡入或硬切闪烁。
- 所有帧统一为 512×512 RGBA 方形画布，运行时仅做等比缩放，修复角色拉伸。
- 抬手、伸展、舞蹈等姿势均为完整角色重绘，消除肩膀白缝、手臂断层、重复手脚和身体长出残片的问题。

### 换装

- 默认、红围巾、蓝披风、圆眼镜和海军帽各自拥有 7 帧完整待机动画。
- 服装从每张待机帧的生成阶段就绘入角色，不再把附件图片贴到 Coco 上。
- 换装只影响待机；互动始终使用默认 Coco，结束后回到所选服装待机。

### 构建与验证

- Windows 将 292 张运行时帧打入单文件 EXE；macOS App 使用同一套整帧资源。
- 新增整帧资源、逐像素端点、独立关键姿势、方形画布和生产渲染路径检查。
- 修正 Windows 与 macOS 产品名称、版本号和应用图标元数据。
- 全面更新架构、素材、构建、测试和故障排查文档。
