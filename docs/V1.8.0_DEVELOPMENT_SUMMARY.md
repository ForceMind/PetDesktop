# v1.8.0 开发总结

## 本次目标

本次开发解决 Windows 滚轮缩放抖动、快速点击中断动画两个直接体验问题，并把同一套 Coco 整帧动画扩展成可独立部署的 Web/PWA 版本。

## 完成内容

### 1. Windows 缩放不再拉伸旧画面

原实现先调用 WinForms `SetBounds` 改变透明分层窗口，再调用 `UpdateLayeredWindow` 提交新位图。两步之间 Windows 会短暂用旧位图填充新尺寸，连续滚轮输入就表现为拉伸、跳动和闪烁。现在渲染器直接把新位图、目标坐标与目标尺寸一次性交给 `UpdateLayeredWindow`，窗口与像素原子更新。滚轮缩放也不再反复弹出会改变窗口宽度的气泡。

### 2. 动作完整播放与单槽队列

Windows、macOS、Web 都使用相同规则：

1. 当前动作播放期间，第一次点击会预排下一动作并显示提示。
2. 再次快速点击只提示队列已满，不覆盖已排动作。
3. 当前动作完整走到第 8 帧。
4. 回到待机首帧并保持约 100ms。
5. 再开始队列动作并显示它自己的对白。

这保留了“动作末帧 = 站立首帧”的连续性契约，也避免用户点击越快动画越混乱。

### 3. 独立 Coco Web Pet

Web 版是纯静态 PWA，无框架构建依赖，直接使用 Canvas 绘制完整角色帧。它包含 32 个动作、七个点击区域、5 套待机换装、拖动与滚轮缩放，同时新增动作点播、随机动作、自动表演/待机/对白开关、四种背景、暂停、全屏、快捷键与设置记忆。对白会根据可用空间自动放在角色左、右或上方。

页面只预载当前待机服装；动作首次使用时才加载，对应帧随后由浏览器和 Service Worker 缓存。GitHub Pages 工作流复用桌面版无损归档，只发布 222 张唯一 PNG 并还原 292 个逻辑帧，不上传约 66.5MB 的重复目录，也不在源码中保存第二份图片。

## 验证

- Windows 单文件构建和无头诊断冒烟测试通过。
- 32 个动作、5 套待机、首尾逐像素一致、整帧渲染与方形画布检查通过。
- Windows/macOS 动作队列与站立交接规则检查通过。
- Windows 分层窗口原子缩放检查通过。
- Web 的 32 动作、5 套换装、资源路径、纯英文对白、状态机、PWA 与 JavaScript 语法检查通过。
- 本地 HTTP 浏览器请求确认网页外壳、JSON、7 张待机帧、图标与 Service Worker 均能正常加载。
