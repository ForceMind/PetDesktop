# Coco 桌宠 v1.7.0 开发总结

## 项目目标

本次开发把用户提供的 `assets/coco.png` 角色形象制作成可直接运行的 Windows 与 macOS 桌面宠物。最终版本覆盖透明无边框窗口、拖动缩放、点击互动、双语对白、鼠标注视、动态待机、换装、应用图标、跨平台构建和 GitHub Releases 自动发布。

## 最终成果

- Windows：单文件 `CocoDesktopPet-Windows.exe`，无需安装，双击运行。
- macOS：Apple Silicon 与 Intel 通用的 `CocoDesktopPet-macOS-universal.zip`。
- 32 种独立互动动作，每种由 8 张完整角色帧组成。
- 5 套待机外观：默认、红围巾、蓝披风、圆眼镜、海军帽，每套 7 张完整帧。
- 点击头部、左右脸、左右手、身体和脚部会从对应动作组中选择效果。
- 支持拖动、滚轮缩放、置顶开关、尺寸菜单、换装和退出。
- 对白气泡自动测量、换行和增高，保持在角色旁边且不遮挡 Coco。

## 动画方案的关键调整

早期方案尝试把头、手臂、腿和服装切成运行时图层，再通过旋转与位移组合动作。这导致肩膀白缝、手臂断层、重复肢体、贴图残片和服装漂浮，也很难保证动作末尾准确回到待机。

最终方案改为“完整角色逐帧动画”：

1. `coco.png` 只作为脸部、麻布材质、纽扣眼睛、牙齿、蓝色羽毛和比例参考，不直接用作待机，因为原图正在招手。
2. 待机首帧重新设计为正面自然站立，双臂放下、双脚着地。
3. 32 个动作分别设计分镜并生成完整 Coco 帧，不再运行时拼接四肢。
4. 每个动作第 1 帧和第 8 帧都逐像素等于默认站立帧，因此结尾可以直接回到待机，不闪烁、不淡入、不重影。
5. 换装从待机帧生成阶段就绘入角色，不是把配件 PNG 贴到身体上。

## 连贯性与画面修复

- 所有生产帧统一为 512×512 RGBA 方形透明画布，运行时只做等比缩放，解决拉伸问题。
- 切帧后使用脚部锚点对齐中间帧，待机水平漂移从约 33 像素收敛至 ±1 像素。
- 清除跨越 3×2 生成网格边缘的头顶、脚底和身体碎片，避免头上冒出其他动作的脚。
- 左右跳、八字移动、太空步、潜行和冲锋使用首尾归零的平滑整帧位移轨迹。
- 292 个逻辑帧无损去重为 222 张 PNG；没有降低分辨率、颜色或透明度，Windows 包保持约 50 MB。

## 待机与自主表演

v1.7.0 不再以 115ms 的速度无限循环点脚：

- 大部分时间保持当前服装的自然站立首帧。
- 每隔约 3–8 秒完整播放一次 0.95 秒待机小动画。
- 启动后约 12–28 秒首次自动表演，此后无操作约 18–36 秒随机表演一次点头、伸展、探头、鞠躬、跳舞等较自然动作。
- 自动动作有 50% 概率显示对白。
- 点击、拖动、滚轮和右键操作会延后计时，拖动过程中不会触发自动动作。

## 语言策略

- 非中文系统固定使用纯英文：标题、菜单、提示、错误和对白都不显示中文，也不显示语言切换。
- 只有系统 UI 语言为中文时，才默认启用中文并显示中文/English 切换。
- 中文模式可以混合少量简单英文；English 模式保持纯英文。
- Windows 使用 `CurrentUICulture` 检测；macOS 使用首选语言检测，并提供简体、繁体应用名称资源。

## 验证与发布

自动检查覆盖：

- 32 个动作的数量、独立中间姿势和逐像素首尾端点。
- 5 套换装待机的完整帧与精确循环端点。
- 方形画布、透明背景、无运行时骨骼/服装叠层。
- 七个点击区域、气泡和离屏启动渲染。
- 英文系统无可见中文、中文系统语言菜单可用。
- 安静站立、待机小动作、超时自动表演和拖动抑制。
- macOS Swift 源码、应用元数据、通用包和资源结构。

代码通过 GitHub Actions 同时构建 Windows 与 macOS，推送 `v*` 标签后自动创建 Release。当前发布页：<https://github.com/ForceMind/PetDesktop/releases/tag/v1.7.0>。

## 当前设计边界

- 换装只影响待机；互动动作使用默认 Coco，动作结束后回到所选服装站立帧。
- macOS 当前使用临时本地签名，首次启动可能需要在 Finder 中右键选择“打开”。
- 历史骨骼和旧动画素材仍保留作研究参考，但生产渲染路径不会加载。
