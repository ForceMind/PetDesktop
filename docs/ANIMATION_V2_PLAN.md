# Coco 整帧动画 v2 设计

## 不可违反的帧约定

- 默认待机第 1 帧是唯一的互动姿势锚点：正面、双臂自然下垂、双脚着地、头部水平。
- 每一帧都是完整渲染的 Coco。运行时不拼接头、手臂、腿、眼睛或服装图层。
- 每个互动动作的第 1 帧和最后一帧都直接复制默认待机第 1 帧，必须逐像素一致。
- 每套换装从生成阶段就是一组完整待机帧，不允许把配件贴在默认角色上。
- 换装只进入待机时间线；互动时间线始终使用默认 Coco。
- 所有生产帧采用固定 512×512 透明画布、统一脚底基线和角色比例。程序只能整体等比缩放。

## 待机设计

每套待机使用 7 帧，第 1 帧与第 7 帧完全相同，中间包含轻微呼吸、眨眼、手脚小动作和自然回落。

运行时不无限循环这 7 帧。角色默认保持第 1 帧自然站立，约每 3–8 秒完整播放一次待机序列；约 18–36 秒没有操作时，才从较自然的互动动作中随机表演一次。这样既保留生命感，也允许 Coco 安静站着。

| ---: | --- | --- |
| 00 | 默认 | 双臂放下，轻呼吸、眨眼、脚尖轻点后站稳 |

| 02 | 蓝披风 | 披风从每帧开始重新绘制并随肩背轻微摆动 |

| 04 | 海军帽 | 帽子从每帧开始重新绘制并跟随头部运动 |

## 验收门槛

1. 默认待机必须自然站立，不能是原参考图的招手姿势。
2. 32 个动作的中间关键帧必须各不相同。
3. 所有互动首尾与默认站立逐像素一致；所有待机循环首尾逐像素一致。
4. 画面不得出现白缝、断肢、重复手脚、帧间闪烁或非等比拉伸。
5. Windows 和 macOS 必须播放同一套整帧资源。
